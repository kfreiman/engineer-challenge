# ADR-003: Выбор API Gateway / Edge Proxy

**Статус:** Proposed  
**Дата:** 2026-03-08  
**Автор:** pi-assistant

## 1. Контекст

Согласно ADR-001 и ADR-002, системе необходим Edge-прокси, который будет выступать единой точкой входа (API Gateway).

**Ограничения:**

- Нативная поддержка Kubernetes (Ingress/CRD).
- Наличие механизма **ForwardAuth** (или External Auth) для интеграции с Ory Kratos/Keto (ADR-001).
- Полноценная поддержка **gRPC** (HTTP/2) для связи с внутренними сервисами (ADR-002).
- Динамическое обновление конфигурации без перезагрузки инстансов.

## 2. Принятое решение

В качестве основного API Gateway выбран **Traefik Proxy**.

## 3. Технические детали и механизмы

- **ForwardAuth Middleware**: Traefik будет делегировать проверку сессии внешнему сервису (Ory Kratos/Oathkeeper), прежде чем проксировать запрос к BFF или бизнес-сервисам.
- **Entrypoints**: Автоматический редирект HTTP (80) на HTTPS (443).
- **Service Discovery**: Использование Kubernetes Ingress или Traefik `IngressRoute` (CRD) для декларативного описания маршрутов.
- **Observability**: Нативная поддержка экспорта метрик в Prometheus и структурированных JSON-логов.

## 4. Рассмотренные альтернативы

### 4.1. NGINX Ingress Controller

- **Плюсы:** Высокая производительность, стандарт де-факто для K8s.
- **Минусы:** Конфигурация через аннотации может быть громоздкой; сложнее настраивать сложные цепочки Middleware.
- **Решение:** Отклонено в пользу более гибкого в конфигурации Traefik.

### 4.2. Envoy / Istio / Gloo

- **Плюсы:** Максимальная гибкость, встроенный Service Mesh.
- **Минусы:** Избыточная сложность (Overhead) и высокий порог вхождения для небольших команд на старте.
- **Решение:** Отклонено как преждевременное усложнение.

## 5. Последствия и риски

### Положительные (Pros)

- Написан на Go (совпадает с основным стеком проекта по ADR-002).
- Встроенный Dashboard для визуального мониторинга состояния роутов.
- Нативная интеграция с Let's Encrypt для автоматического получения SSL-сертификатов.

### Отрицательные / Риски (Cons/Risks)

- Производительность на крайне высоких нагрузках может уступать Envoy/NGINX.
- **Митигация:** Горизонтальное масштабирование реплик Traefik в K8s и использование Cloud Load Balancer перед ним.
